문제 : 5397번 '키로거' (난이도 실버3)
풀이시간 : 50분 추정
한줄평 : 알고리즘상으로는 전혀 문제가 없었으나, for문에 strlen이 들어간 것이 시간초과의 문제가 되었다. 조건식을 비교할 때 마다 strlen 즉, 문자열의 길이를 구하기 위해 자체적으로 반복을 했기 때문에, 실질적으로 상당한 연산을 하게 되었다. strlen값을 담은 변수를 따로 선언한 뒤 for문의 조건식에 넣으니 시간초과 없이 문제를 풀 수 있었다. 앞으로 strlen함수는 사용을 최대한 자제하는게 좋을 것 같다.
 
 --------알고리즘--------
 
[핵심 아이디어]
스택 2개를 만들어서 각각 커서 왼쪽, 오른쪽을 담당한다.
>는 왼쪽스택에서 오른쪽으로, <는 오른쪽에서 왼쪽스택으로 하나씩 옮긴다.
-는 왼쪽스택의 top에서 제거한다.
 
[알고리즘 흐름]
1.문자열 input을 입력받는다. (input[1000100])
2.문자 하나하나씩 보면서 다음을 실행한다.
 2.1. 문자가 <인 경우 rightStack[++rightTop] = leftStack[leftTop--] 단, leftTop == -1인 경우 무시
 2.2. 문자가 >인 경우 leftStack[++leftTop] = rightStack[rightTop--] 단, rightTop == -1인 경우 무시
 2.3. 문자가 -인 경우 leftTop-- 단, leftTop == -1인 경우 무시
 2.4. 그 외에는 leftStack[++leftTop]에 문자 저장
3. leftStack[0~leftTop], rightStack[rightTop ~ 0]을 출력
